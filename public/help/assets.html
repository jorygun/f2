Help on Assets

"Assets" are the various pictures, documents, and audio/video items that have been uploaded to the site over the years. The assets have been (mostly) indexed in a database for easy search and retrieval. Use this page to find and retrieve an asset.

Graphic assets are stored at the original resolution and also one or more fixed size 'thumbnails'. Thumbnails are used to display the asset in news articles and elsewhere. Clicking on a thumbnail retrieves the original image. .

Assets have various attributes, including a title, caption, vintage, and keywords, a "source" - the original document referenced, and a thumbnail image - a small graphic to represent the source.

Asset Data

Top Table Contains some useful information: Status, Date/time entered, mime-type and corresponding type, size, and first use. If the first use is recorded, you can click on the link to view it.
Item title Title for the asset. Searchable. Limited to 64 characters.
Caption Caption, especially names of people in a picture. Searchable. Can be as long as you need within reasonable limits.
Status (admin only) Set status to various codes for reviewed, errors.
Origin Enter the guestimated year this item was created and an attribution to a person or organization or publication.
Contributor Name and id of the person contributing this item. Usually autoomatically set based on logged in user. User name must exactly match Flames user name. (id will be set automatically from the name.)
Tags Tags are another way to categorize an archival asset. Tags are completely optional.
Source URLThis is the url of the document this asset represents. It can be a local file ('/assets/files/4566.php') or a remote web page. (Remote web pages may not be reliable over a long term.) Youtube videos are often used.
Thumb Source Normally you can leave this blank and the thumbnail image for the asset is created automatically from the source document. The always works for images, pdf files, and youtube videos. For other documents, a generic icon might be used. But the uploader can manually specify another source for the thumbnail. For example another different graphic can be specified, or a graphic file uploaded. This is especially useful for video assets that aren't on youtube, so a useful thumb can't be automatically created. The Thumb URL should always be stored locally, not a remote web page.
Notes Uploader notes. Saved but not displayed or searchable. Often contains the name of the original source that was uploade, like 'image_005.jpg'.
Thumb Images Thumbs may exist in 3 different sizes: Thumb (~200px), Gallery (~350px), or Toon (~800px). Every asset has a thumb image. Gallery are larger ones used in gallery displays. Toon is page width size and used for cartoons at top of news or full-width displays in articles ('article-main'). This panel shows you which sizes exist and lets you check to create or recreate.

The normal thumbs is displayed under the editor form. Sometimes this image does not get updated on your browser after a change, due to caching. If in doubt, right-click on the image and choose to open image in a new tab, and then you can refresh it. Usually. Sometimes.
